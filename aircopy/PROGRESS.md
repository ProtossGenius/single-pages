## 当前进度

### 阶段 1：基础连通（已完成）

- [x] 连接前页面：二维码 / 扫描框双模式，支持移动端优先扫码、桌面端优先二维码。
- [x] 展示名输入与默认值。
- [x] Web 端扫码流程与扫码特效。
- [x] 连接后进入聊天界面。
- [x] 首条问候从 `hello world` 升级为 `hello 彼此名字`。
- [x] 二维码支持双击放大与全屏查看。

### 阶段 1.5：连通方案调整（已完成）

- [x] 从“长 SDP 二维码交换”切换为 `PeerJS + 短 peerId 二维码`，显著缩短二维码内容长度。
- [x] 保留扫码主链路（ZXing），增加黑色区域候选识别兜底（jsQR）。

### 阶段 2：普通聊天（进行中）

- [x] 基础文本发送与接收。
- [x] 连接对象名称显示（会话标题/会话列表）。
- [x] 消息时间戳（HH:MM）。
- [x] 未读提示（标题/会话列表角标）。
- [x] 会话状态显示（在线/离线）。
- [x] 手机端会话抽屉 + 桌面端会话栏显隐切换。
- [x] 会话持久化（刷新后保留历史）。
- [x] 聊天页新增“连接更多”入口，可回到二维码/扫码页继续建链。
- [x] 当前会话可见时，未读自动清零（修复首次连接未读残留）。
- [x] 持久化改为基于双方交换的稳定 `persistentId`（不再依赖易变 `peerId`）。

### 阶段 3：多连接会话管理（规划中）

- [ ] 任意节点可同时维护多个点对点会话（例如多个端连接同一个端点 C 时，C 为每个连接维护独立会话）。
- [ ] 左侧会话列表中的每一项对应一个对端节点，会话可切换查看历史与继续聊天。
- [ ] 左侧会话支持删除，但不能删除当前正在聊天的会话；删除时清理本地会话记录。

> 说明：原“阶段 3：群聊”和“阶段 4：中间人转发私聊”方案已取消，整体改为多会话点对点模型，不再实现群聊/中间人模式。

### 阶段 5：聊天能力增强（单对单已完成）

- [x] 表情（单对单：输入区表情面板 + 插入发送）。
- [x] 文件传输（单对单：DataChannel 分片发送 + 接收下载）。
- [x] 语音（单对单：MediaRecorder 录音发送 + 接收播放）。
- [x] 视频通话（单对单：PeerJS MediaConnection 发起/接听/挂断）。
