# AI 小说写作工具 — 需求文档

> 版本: 2.0  
> 最后更新: 2026-03-16

---

## 1. 项目概述

本项目是一个基于浏览器的 AI 辅助小说写作工具，作为 `single-pages` 项目的子目录独立运行。用户可以管理小说世界观设定（人物、地点、宗门等），使用 AI 辅助生成小说段落和章节，并进行导入/导出操作。

## 2. 技术约束

| 约束项 | 说明 |
|--------|------|
| 技术栈 | 纯 HTML + CSS + JavaScript，无构建工具 |
| 独立性 | 不依赖父目录及其他子项目的代码 |
| 第三方库 | 如需使用，下载到本地 `vendor/` 目录 |
| 数据持久化 | 使用浏览器 IndexedDB |
| AI 接口 | 仅支持 OpenAI 兼容接口 |

## 3. 功能需求

### 3.1 类目管理器（左侧面板）

#### 3.1.1 基本功能
- **FR-CAT-001**: 支持创建、编辑、删除类目大类（如：人物、地点、宗门等）
- **FR-CAT-002**: 每个大类下支持创建具体成员
- **FR-CAT-003**: 每个成员下可以有无限层级的子类（树形结构）
- **FR-CAT-004**: 选中某一项时，在类目管理器右侧增加一列，显示该项的详细信息
- **FR-CAT-005**: 详细信息支持编辑和保存

#### 3.1.2 数据存储要求
- **FR-CAT-010**: 每个项目占一行数据
- **FR-CAT-011**: 主键使用 UUID
- **FR-CAT-012**: 用 `parentId` 字段记录父 UUID（顶层项为 null）
- **FR-CAT-013**: 用 `version` 字段表明版本号
- **FR-CAT-014**: 用 `updatedAt` 字段记录最后修改时间（精确到毫秒）

### 3.2 菜单栏（顶部）

#### 3.2.1 文件菜单
- **FR-FILE-001**: 文件导入 — 接受压缩文件(.zip)，导入后数据持久化到 IndexedDB
- **FR-FILE-002**: 文件导出 — 将工作区所有数据打包为 .zip 文件下载，显示压缩进度
- **FR-FILE-003**: 导入/导出一致性 — 配置 → 导出 → 导入后数据应与原始配置完全一致

#### 3.2.2 AI 配置菜单
- **FR-AI-001**: AI 供应商配置 — 支持配置多个 AI 供应商
- **FR-AI-002**: 每个供应商可配置多个模型
- **FR-AI-003**: 支持配置重试次数
- **FR-AI-004**: 目前仅支持 OpenAI 兼容接口

#### 3.2.3 职能配置
- **FR-ROLE-001**: 职能（职位）是一个固定枚举
- **FR-ROLE-002**: 每个职能有对应的提示词模板
- **FR-ROLE-003**: 提示词支持使用变量枚举占位（如：`{{前文信息}}`、`{{用户输入}}` 等）
- **FR-ROLE-004**: 每个职能需配置使用的 AI 供应商及模型
- **FR-ROLE-005**: 每个职能有一个输出内容变量枚举（与提示词变量是同一套枚举）

#### 3.2.4 流程配置
- **FR-FLOW-001**: 配置数据传输流程
- **FR-FLOW-002**: 有触发方式（枚举，如：生成章节、历史配置改动等）
- **FR-FLOW-003**: 流程体是一个二维数组 `[[职位]]`
  - 内层数组中的职位并行执行
  - 外层数组中的元素依次执行
- **FR-FLOW-004**: 支持启用/禁用流程
- **FR-FLOW-005**: 支持配置是否阻塞
  - 如果输出枚举标记为阻塞，则强制阻塞
  - 如果输出枚举标记为非阻塞，由用户选择
  - 阻塞时，"开始生成"、"生成章节"等按钮不可点击

### 3.3 小说编写界面（中间区域）

#### 3.3.1 小说内容区（上部）
- **FR-NOVEL-001**: 顶部显示章节名，可编辑
- **FR-NOVEL-002**: 正文按段落分割显示
- **FR-NOVEL-003**: 每个段落可单独选中
- **FR-NOVEL-004**: 每个段落绑定以下信息：
  - 涉及的人物、物品等类目信息（有版本）
  - 前文提要（无版本，单独一行数据）
  - 后续内容（无版本，单独一行数据）
- **FR-NOVEL-005**: 段落记录最后更新时间（精确到毫秒）

#### 3.3.2 聊天框（下部）
- **FR-CHAT-001**: 可折叠的状态栏，显示 AI 运行状态
  - 格式: `主流程 N/N 次流程 N/N {当前职能名}`
  - 点击展开：显示各二级流程执行时长、失败次数/重试次数
  - 失败原因可点击复制到剪贴板
- **FR-CHAT-002**: 输入框，通过 Tab 切换三种模式：
  - 章节概述（文本框）
  - 后续概要（文本框）
  - 绑定设定（引用左侧类目管理器的设定信息）
- **FR-CHAT-003**: 底部按钮：
  - "开始生成" — 生成段落
  - "生成章节" — 生成整章概要并保存
  - 有运行中的 AI 任务时按钮置灰不可点击

### 3.4 当前状态面板（最右侧）
- **FR-STATUS-001**: 显示章节概要
- **FR-STATUS-002**: 显示 AI 评审意见
- **FR-STATUS-003**: 显示其他章节信息

### 3.5 前情提要生成
- **FR-RECAP-001**: 前情提要 = 所有已完成章节的概述 + 当前章节已写部分
- **FR-RECAP-002**: 采用阶梯形压缩策略，越早的内容占字越少
- **FR-RECAP-003**: 压缩参数可配置：
  - 每次处理的章节数（默认 10）
  - 每次压缩的目标字数（默认 500）
- **FR-RECAP-004**: 生成算法：
  1. 取前 N 章内容生成约 500 字前情提要
  2. 将此提要与后 N 章概要合并，再生成 500 字概要
  3. 重复直至覆盖所有已完成章节
- **FR-RECAP-005**: 每完成一章，用章节概要与已有前情概要融合一次

### 3.6 章节管理
- **FR-CHAP-001**: 章节内容存储在独立的表中，主键递增
- **FR-CHAP-002**: 导出时可选择单独导出小说内容
- **FR-CHAP-003**: 每章保存时同时保存章节概要

## 4. 非功能需求

- **NFR-001**: 数据完全存储在浏览器本地（IndexedDB），无服务端
- **NFR-002**: 页面加载后可离线使用（AI 功能除外）
- **NFR-003**: 支持现代浏览器（Chrome, Firefox, Edge 最新版本）
- **NFR-004**: UI 响应时间 < 200ms（不含 AI 请求）

## 5. V1 版本范围限制

- 不支持修改历史数据
- 仅支持以下核心流程：
  1. 用户点击"开始生成" → 根据段落概述、后续概要、绑定设定生成新段落
  2. 用户点击"生成章节" → 为整章内容生成概要，与章节内容一起保存到章节表

## 6. 测试要求

- **TEST-001**: AI 配置部分测试不使用真实 AI，为每组数据生成随机数据
- **TEST-002**: 文件导入/导出测试：配置 → 导出 → 导入 → 验证数据一致性
- **TEST-003**: 每个基础单元需有对应的测试代码

## 7. V2 新增需求

> 以下为 V2 版本新增的功能需求，在 V1 (P0-P7) 全部完成后实施。

### 7.1 书籍管理

- **FR-BOOK-001**: 新增书籍管理页面，用户可创建/选择/删除书籍
- **FR-BOOK-002**: 所有章节、段落、绑定等数据归属于某一本书
- **FR-BOOK-003**: 同一时间只能编辑一本书，通过书籍管理页面切换
- **FR-BOOK-004**: 新增书籍时可设置书名和简介

### 7.2 左侧边栏重构

- **FR-SIDE-001**: 左侧边栏改为 VSCode 风格，最左边一列图标 Tab 按钮
- **FR-SIDE-002**: 图标 Tab 包含三个页面：
  - 类目设定（原有的类目管理器 + 详情面板合并）
  - 章节目录（章节列表，可切换章节）
  - 书籍信息（书名、简介等元信息）
- **FR-SIDE-003**: 点击不同 Tab 切换左侧面板内容
- **FR-SIDE-004**: 中央小说内容区根据左侧边栏的选择动态变化：
  - 选择「类目设定」中的某项时，内容区显示该类目的详细信息（可编辑）
  - 选择「章节目录」中的某章时，内容区显示该章的段落编辑器
  - 选择「书籍信息」时，内容区显示书籍元信息编辑

### 7.3 菜单重命名

- **FR-MENU-001**: 文件菜单中的 "导入..." 改为 "导入工作区..."
- **FR-MENU-002**: 文件菜单中的 "导出..." 改为 "导出工作区..."

### 7.4 AI 供应商配置增强

- **FR-AICFG-001**: 供应商配置页面新增 API 测试按钮
- **FR-AICFG-002**: 点击测试按钮弹出测试面板，包含：
  - 问题输入框（用户输入测试问题）
  - 预览区（显示 AI 回复）
  - 错误信息区（红色显示错误）
- **FR-AICFG-003**: 测试按钮使用当前页面配置的 API 地址、Key、选中的模型进行实际 API 调用
- **FR-AICFG-004**: 支持 Ollama 特有的响应格式处理
  - Ollama 的 `/api/chat` 接口返回格式与 OpenAI 有差异
  - 自动检测并适配 Ollama 的响应结构

### 7.5 模型智能等级

- **FR-LEVEL-001**: 每个 AI 模型可配置一个智能等级（枚举值）
- **FR-LEVEL-002**: 智能等级用于模板导入时的自动匹配
- **FR-LEVEL-003**: 等级枚举：高级、中级、基础

### 7.6 职能系统重构

- **FR-ROLE2-001**: 移除固定的 RoleEnum，所有职能均由用户自行创建
- **FR-ROLE2-002**: 职能配置使用 UUID 作为主键，而非枚举值
- **FR-ROLE2-003**: 用户可自由 CRUD 职能（添加、编辑、删除）
- **FR-ROLE2-004**: 流程配置中的步骤引用职能的 UUID

### 7.7 模板导入/导出

- **FR-TPL-001**: 支持将职能配置 + 流程配置导出为 JSON 模板文件
- **FR-TPL-002**: 导入模板时，根据模型智能等级自动匹配本地已有模型
  - 导出时记录每个职能使用的模型智能等级
  - 导入时按智能等级匹配本地模型（同等级优先，无匹配时提示用户手动选择）
- **FR-TPL-003**: 模板文件包含所有职能的提示词、变量绑定、流程步骤定义

### 7.8 聊天框重构

- **FR-CHAT2-001**: 移除 Tab 切换结构，改为垂直可折叠区块
- **FR-CHAT2-002**: 将 "章节概述" 重命名为 "情节概述"
- **FR-CHAT2-003**: 每个区块（情节概述、后续概要、绑定设定）独立折叠/展开
- **FR-CHAT2-004**: 新增 "直接添加" 按钮，将输入内容直接作为新段落添加（不经过 AI）
- **FR-CHAT2-005**: 新增 AI 风格标签区域
  - AI 生成时可返回风格标签（如：悬疑、热血、日常等）
  - 标签显示在聊天框区域，用户可编辑/删除
  - 风格标签作为额外上下文传递给 AI

### 7.9 大纲导入

- **FR-OUTLINE-001**: 聊天框区域新增 "大纲导入" 可折叠区块
- **FR-OUTLINE-002**: 支持粘贴/输入长文本大纲
- **FR-OUTLINE-003**: 系统逐行处理大纲文本，每行作为一个情节概述
- **FR-OUTLINE-004**: 处理方式：将每行作为 "情节概述" 依次调用 AI 生成段落

### 7.10 自定义变量

- **FR-CVAR-001**: 职能配置中支持定义自定义变量
- **FR-CVAR-002**: 自定义变量为流程级作用域，在流程执行期间存在，执行完毕后清除
- **FR-CVAR-003**: 自定义变量不持久化到数据库
- **FR-CVAR-004**: 自定义变量支持依赖检查：流程引擎检查步骤中使用的变量是否已被前序步骤产出
- **FR-CVAR-005**: 在提示词模板中可使用 `{{自定义:变量名}}` 格式引用自定义变量

### 7.11 日志管理

- **FR-LOG-001**: 持久化所有 AI 请求和响应日志到 ai_logs 表
- **FR-LOG-002**: 日志包含：请求时间、供应商、模型、提示词、响应内容、耗时、状态（成功/失败）、错误信息
- **FR-LOG-003**: 新增日志浏览页面（模态对话框），可按时间倒序浏览
- **FR-LOG-004**: 支持导出日志为 JSON 文件
- **FR-LOG-005**: 支持自动清理（按天数或条数限制）

### 7.12 AI 提示词自优化

- **FR-PROMPT-001**: 新增 AI 提示词自优化功能
- **FR-PROMPT-002**: 用户可选择一个高级模型作为优化器
- **FR-PROMPT-003**: 优化器分析现有提示词模板，给出改进建议
- **FR-PROMPT-004**: 用户可预览优化建议并选择性应用
- **FR-PROMPT-005**: 优化结果可与原始模板对比展示
