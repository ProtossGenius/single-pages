## 2. 流程一：生成段落

### 2.1 触发条件
- 用户点击 "开始生成" 按钮
- 前提：无正在运行的阻塞 AI 任务

### 2.2 流程步骤

```
用户点击 "开始生成"
    │
    ▼
┌─────────────────────────────┐
│ 1. 收集上下文信息           │
│    - 获取用户输入(章节概述)  │
│    - 获取后续概要            │
│    - 获取绑定的设定信息      │
│    - 生成前文信息            │
│      (调用前情提要生成逻辑)   │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 2. 查找匹配的流程配置       │
│    - trigger == 'generate_  │
│      paragraph'             │
│    - enabled == true        │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ 3. 按流程配置执行 (二维数组)                │
│                                             │
│    步骤1: [写手, 评审员]  → 并行执行        │
│      │                                      │
│      ▼                                      │
│    步骤2: [概要师]        → 串行执行        │
│                                             │
│    每个职能:                                │
│    a. 取 role_configs 获取提示词模板         │
│    b. 替换变量占位符为实际值                │
│    c. 调用 AI API (含重试逻辑)              │
│    d. 将输出写入对应的输出变量              │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 4. 处理输出                 │
│    - generated_paragraph    │
│      → 创建新段落           │
│    - ai_review              │
│      → 更新右侧状态面板     │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 5. 更新 UI                  │
│    - 在小说内容区添加新段落  │
│    - 更新状态栏             │
│    - 恢复按钮可点击状态     │
└─────────────────────────────┘
```

### 2.3 变量传递机制

在流程执行过程中，维护一个变量上下文 (Context) 对象，格式如下：

```javascript
{
  context_before:      "",   // 前文信息
  user_input:          "",   // 用户输入(章节概述)
  chapter_outline:     "",   // 章节概述
  follow_up:           "",   // 后续概要
  bound_settings:      "",   // 绑定设定(序列化文本)
  current_paragraph:   "",   // 当前段落内容
  chapter_content:     "",   // 章节内容
  ai_review:           "",   // AI评审意见(可作为输入和输出)
  generated_paragraph: "",   // 生成段落(输出)
  generated_summary:   "",   // 生成概要(输出)
  generated_recap:     "",   // 生成提要(输出)
}
```

- **输入变量**: 在流程开始前从各数据源填充
- **输出变量**: 在职能执行完成后由 AI 返回值写入
- 后续步骤可以读取前面步骤写入的输出变量

## 3. 流程二：生成章节

### 3.1 触发条件
- 用户点击 "生成章节" 按钮
- 前提：无正在运行的阻塞 AI 任务
- 前提：当前章节至少有一个段落

### 3.2 流程步骤

```
用户点击 "生成章节"
    │
    ▼
┌─────────────────────────────┐
│ 1. 收集当前章节信息         │
│    - 合并所有段落为完整文本  │
│    - 获取章节标题           │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 2. 查找匹配的流程配置       │
│    - trigger == 'generate_  │
│      chapter'               │
│    - enabled == true        │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│ 3. 按流程配置执行 (二维数组)                │
│    通常: [概要师] 生成章节概要              │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 4. 保存章节                 │
│    - 将内容和概要写入       │
│      chapters 表            │
│    - 状态改为 'completed'   │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 5. 更新前情提要             │
│    - 用新章节概要与已有     │
│      前情概要进行融合       │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 6. 开始新章节               │
│    - 创建新的章节记录       │
│    - 清空编辑区             │
│    - 更新 UI                │
└─────────────────────────────┘
```

## 4. 流程三：前情提要生成

### 4.1 阶梯形压缩算法

配置参数：
- `N`: 每次处理的章节数 (默认 10)
- `W`: 每次压缩的目标字数 (默认 500)

### 4.2 初始化生成（历史章节多时）

```
章节: [C1, C2, C3, ..., C30, C31(当前)]

第一轮: 取 C1~C10 的概要 → AI生成约500字提要 → R1
第二轮: R1 + C11~C20 的概要 → AI生成约500字提要 → R2
第三轮: R2 + C21~C30 的概要 → AI生成约500字提要 → R3

最终前情提要 = R3 + 当前章节已写部分
```

### 4.3 增量更新（每完成一章时）

```
已有前情提要: R_prev
新完成章节概要: S_new

调用 AI:
  输入: R_prev + S_new
  要求: 融合为不超过 W 字的前情提要
  输出: R_new

保存 R_new 到 recap_data 表
```

### 4.4 流程图

```
┌──────────────────────┐
│ 需要生成前情提要     │
└──────────────────────┘
    │
    ▼
┌──────────────────────┐
│ 是否有已保存的       │──── 有 ────→ 使用已保存的最新提要
│ recap_data?          │              作为 R_prev
└──────────────────────┘
    │ 无
    ▼
┌──────────────────────┐
│ 获取所有已完成章节   │
│ 按顺序分组(每N章)    │
└──────────────────────┘
    │
    ▼
┌───────────────────────────────┐
│ 循环: 对每组章节              │◄───┐
│   取当前组的章节概要          │    │
│   与前一组的提要合并          │    │
│   调用 AI 生成 ~W 字的提要   │    │
│   保存到 recap_data          │────┘
└───────────────────────────────┘
    │
    ▼
┌──────────────────────┐
│ 返回最终的前情提要   │
│ = 最后一轮提要       │
│ + 当前章节已写内容   │
└──────────────────────┘
```

## 5. 流程四：文件导出

### 5.1 流程步骤

```
用户点击 "导出"
    │
    ▼
┌─────────────────────────────┐
│ 1. 显示导出进度对话框       │
│    进度: 0%                 │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 2. 逐表读取数据             │
│    每读完一个表更新进度      │
│    (10个表, 每个10%)        │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 3. 生成 manifest.json       │
│    记录各表数据条数          │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 4. 使用 JSZip 打包为 .zip   │
│    更新进度条               │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 5. 触发浏览器下载           │
│    关闭进度对话框           │
└─────────────────────────────┘
```

### 5.2 进度计算

| 阶段 | 进度范围 | 说明 |
|------|----------|------|
| 读取数据 | 0% - 70% | 每张表占 7% |
| 生成 manifest | 70% - 75% | |
| 压缩打包 | 75% - 95% | |
| 触发下载 | 95% - 100% | |

## 7. AI 调用流程

### 7.1 单次 AI 调用

```
┌────────────────────────────────────┐
│ AI 调用 (role, context)            │
└────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────┐
│ 1. 获取职能配置 (role_configs)     │
│    - 提示词模板                    │
│    - 供应商 & 模型                 │
└────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────┐
│ 2. 替换变量占位符                  │
│    {{前文信息}} → context.         │
│    context_before                  │
│    {{用户输入}} → context.         │
│    user_input                      │
│    ...                             │
└────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────┐
│ 3. 获取供应商配置 (ai_providers)   │
│    - API URL                       │
│    - API Key                       │
│    - 重试次数                      │
└────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────┐
│ 4. 调用 OpenAI 兼容 API                     │
│    POST {apiUrl}/v1/chat/completions         │
│    {                                         │
│      model: modelName,                       │
│      messages: [                             │
│        { role: "system", content: prompt }   │
│      ]                                       │
│    }                                         │
└──────────────────────────────────────────────┘
    │
    ├── 成功 ──→ 返回 AI 结果, 写入 context 的输出变量
    │
    └── 失败 ──→ 记录错误信息
                  │
                  ├── 重试次数未用完 → 重试 (回到步骤4)
                  │
                  └── 重试用完 → 标记失败, 记录错误原因

```

### 7.2 状态更新

AI 调用过程中持续更新 UI 状态栏：

```javascript
// 状态对象结构
{
  flowName:    "生成段落流程",
  totalSteps:  3,
  currentStep: 2,
  steps: [
    {
      index: 0,
      roles: [
        {
          role: "writer",
          displayName: "写手",
          status: "completed",   // 'pending' | 'running' | 'completed' | 'failed'
          duration: 2300,        // 已运行毫秒数
          failCount: 0,
          maxRetry: 3,
          error: null
        }
      ]
    },
    // ...
  ]
}
```

## 8. 类目管理流程

### 8.1 添加类目项

```
用户点击 [+ 添加]
    │
    ▼
显示输入框 (名称)
    │
    ▼
用户输入名称并确认
    │
    ▼
生成 UUID → 创建 category 记录:
  - id: newUUID
  - parentId: 当前选中节点的 id (或 null)
  - version: 1
  - updatedAt: Date.now()
    │
    ▼
保存到 IndexedDB
    │
    ▼
刷新树形视图
```

### 8.2 修改类目项

```
用户编辑详情面板中的内容并点击 [保存]
    │
    ▼
version + 1
updatedAt = Date.now()
    │
    ▼
更新 IndexedDB 中的记录
    │
    ▼
刷新树形视图和详情面板
```

### 8.3 删除类目项

```
用户点击 [删除]
    │
    ▼
确认对话框: "确定删除此项及其所有子项？"
    │
    ├── 取消 → 关闭
    │
    └── 确认
        │
        ▼
    递归查找所有子节点 ID
        │
        ▼
    删除所有子节点 + 自身
    删除关联的 paragraph_bindings
        │
        ▼
    刷新 UI
```
