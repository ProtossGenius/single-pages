## 1. 整体布局

> V2 变更: 布局从四栏式改为 VSCode 风格。左侧为图标导航栏+侧边面板，中间为动态内容区+聊天框，右侧为状态面板。

```
┌────┬───────────────┬──────────────────────────────────┬──────────────┐
│    │               │            菜单栏                │              │
│图标│               ├──────────────────────────────────┤              │
│导航│   侧边面板    │                                  │   当前状态   │
│栏  │  (根据Tab     │      动态内容区                  │    面板      │
│    │   切换内容)   │  (根据选择显示不同内容)           │              │
│ 📋 │               ├──────────────────────────────────┤              │
│ 📁 │               │        聊天框区域                │              │
│ 📖 │               │  (状态栏+可折叠区块+按钮)        │              │
└────┴───────────────┴──────────────────────────────────┴──────────────┘
```

### 1.1 尺寸设计

| 区域 | 宽度 | 说明 |
|------|------|------|
| 图标导航栏 | 48px (固定) | V2: 最左边一列图标按钮 |
| 侧边面板 | 300px (可拖拽调整) | V2: 替代原有类目管理器+详情面板 |
| 动态内容区 | flex: 1 (自适应) | V2: 根据侧边栏选择显示不同内容 |
| 当前状态面板 | 280px (可拖拽调整) | 最小 200px，最大 400px |

### 1.2 菜单栏
- 位置：小说编写区和当前状态面板的顶部（横跨中间和右侧两栏）
- 高度：40px
- 包含菜单项：文件、AI 配置

## 2. 类目管理器（左侧面板）

### 2.1 树形结构

```
┌─────────────────────────┐
│ 📂 类目管理器      [+]  │
│─────────────────────────│
│ ▶ 人物                  │
│   ├── 主角              │
│   │   ├── 张三          │
│   │   └── 李四          │
│   └── 反派              │
│       └── 王五          │
│ ▶ 地点                  │
│   ├── 修仙界            │
│   └── 凡人界            │
│ ▶ 宗门                  │
│   ├── 天剑宗            │
│   └── 万法宗            │
│ ▶ 物品                  │
│   └── 神兵              │
└─────────────────────────┘
```

### 2.2 交互说明
- 点击 `▶` / `▼` 展开/折叠子节点
- 点击节点文本选中该节点
- 选中节点高亮显示
- 每个节点右侧显示操作按钮（hover时显示）：
  - `+` 添加子节点
  - `✏️` 编辑
  - `🗑️` 删除
- 顶部 `[+]` 按钮添加新的顶级大类
- 支持右键菜单（添加子项、编辑、删除）

- 点击段落选中，显示高亮边框

## 8. 响应式设计

- 最小支持宽度: 1200px
- 低于 1200px 时，侧边面板和当前状态面板可通过按钮切换显示/隐藏
- 弹窗（模态对话框）始终居中显示，宽度自适应（最大 800px）

## 9. V2 新增 UI 设计

### 9.1 左侧图标导航栏 (VSCode 风格)

```
┌────┬────────────────────────────┐
│    │                            │
│ 📋 │   类目设定面板内容         │  ← 当前选中的Tab
│    │   (原类目管理器+详情面板)  │
│ 📁 │                            │
│    │                            │
│ 📖 │                            │
│    │                            │
└────┴────────────────────────────┘
```

**图标 Tab:**
| 图标 | 说明 | 对应侧边面板内容 |
|------|------|------------------|
| 📋 | 类目设定 | 类目树形视图 (原有功能) |
| 📁 | 章节目录 | 当前书籍的章节列表 |
| 📖 | 书籍信息 | 当前书籍的元信息 |

**交互:**
- 点击图标切换侧边面板内容
- 选中的图标高亮显示（左侧边框蓝色高亮）
- 同时中央内容区也根据选择变化

**优化职能提示词:**
- 用户意见输入框: 用户输入对当前提示词的改进建议
- [优化职能提示词] 按钮: 调用高级模型，传入以下信息:
  1. 原始提交的提示词（变量展开前 + 展开后）

  1. 新的提示词模板

### 9.18 修复: 类目设定侧边栏加号按钮

**问题**: 侧边栏类目设定的 `+` 按钮点击无反应。  
**原因**: 按钮触发 `sidebar:add-root-category` 事件，但 CategoryUI 未监听该事件。  
**修复**: 在 `CategoryUI.init()` 中添加 `EventBus.on('sidebar:add-root-category', () => addCategory(null))`。

### 9.19 更新设定: 树结构 + 新增支持

**变更**:
- AI 分析提示词中仅提供设定的**树结构**（名称、父子关系、类型），不包含具体描述和属性内容
- 分析结果区分「更新现有设定」和「新增设定」两种操作
- 新增设定时通过 parentName 或 parentId 确定父节点

**AI 返回格式**:
```json
[
  { "action": "update", "categoryId": "id", "categoryName": "名称", "changes": { ... } },
  { "action": "add", "parentName": "父节点名", "name": "新设定名", "type": "character", "description": "...", "attributes": { ... } }
]
```

### 9.20 修复: 生成后内容不更新

**问题**: 点击「开始生成」后，AI 返回了数据但小说内容未更新，按钮灰色不可用。  
**原因**: FlowEngine 执行完成后，`context.generated_paragraph` 已填充，但 `handleGenerate()` 未将其创建为新段落。  
**修复**:
- `handleGenerate()` 完成后取 `context.generated_paragraph` 调用 `EditorUI.addParagraph()` 创建新段落
- 同时更新右侧状态面板（概要、评审意见）
- FlowEngine 中增加 `console.log` 打印当前步骤名和下一步骤名

### 9.21 修复: 章节目录侧边栏多余加号

**问题**: 章节目录左边栏的 `+` 按钮点击后导致数据重复渲染。  
**修复**: 移除 `showChapterPanel()` 中的 `+` 按钮。新章节通过菜单或其他方式创建。

### 9.22 日志: 记录变量名和变量值

**变更**: AI 日志中增加 `variables` 字段，记录流程执行时所有模板变量的名称和对应值。

**数据格式**:
```json
{
  "variables": [
    { "name": "章节概述", "value": "张三在山洞修炼..." },
    { "name": "绑定设定", "value": "[人物>张三] 年龄18..." }
  ]
}
```

### 9.23 提示词优化功能完善

**变更**:
1. **新提示词可编辑**: 优化结果中的新提示词使用 `<textarea>` 代替 `<pre>`，用户可手动修改后再应用
2. **清空用户意见**: 生成新提示词后自动清空用户意见输入框
3. **用原始模型生成**: 新增按钮，使用职能配置的原始模型 + 最新提示词（role.promptTemplate）生成内容
4. **高级AI生成使用最新提示词**: 「用高级AI生成内容」按钮使用 role.promptTemplate 的最新版本构建提示词，而非日志中的历史提示词
