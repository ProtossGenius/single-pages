本项目是由html、css、js来实现的，每个子目录都是一个独立的页面，尽可能做到让它不以来父目录及其他项目的代码（如果有js，就下载到本地）

工作时将整体工作拆分成若干可验证的子任务，整体工作先拆分成一份计划文档，确保文档中包含了所有开发的基础要求
## 每个子任务的工作流
1. 实现子功能
2. 如果是基础单元，添加测试代码到测试页面
3. 运行测试
4. 确认全部测试通过
5. 更新计划文档中对应任务的 checkbox
6. git commit


目前项目所在的目录是writer目录，其中docs里已经包含了所需要的文档。这个任务同样需要先更新文档，然后再继续工作。有一个服务基于项目的根目录提供了https访问
